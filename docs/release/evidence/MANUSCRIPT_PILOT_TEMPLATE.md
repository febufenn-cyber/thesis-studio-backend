# Manuscript Pilot — Per-Document Evidence Template

> **Status note:** Real institutional manuscripts remain **outstanding**. To date the
> pilot harness has only been run against **synthetic fixtures** generated by
> `scripts/generate_pilot_fixtures.py` (raw results in
> `docs/release/evidence/PILOT_SYNTHETIC_RAW.json`). This template must be completed
> once per real, anonymised/permissioned manuscript before the pilot threshold in
> `docs/release/REAL_MANUSCRIPT_PILOT.md` can be considered met. Never place
> identifiable student manuscripts in the repository or CI artifacts.

Copy this file to `docs/release/evidence/pilot/<PILOT_ID>.md` for each document.
Automated fields can be copied from the per-file JSON emitted by
`scripts/run_manuscript_pilot.py`; human fields must never be auto-filled — they
stay `pending-human-review` until a named person completes them.

## Notes for the operator

- `Upload malware result` in local harness runs reads `disabled`; a real pilot on
  staging/production infrastructure must run with ClamAV enabled and record the
  actual scan result.
- `Reviewer identity` and all visual-comparison fields are human attestations.
  The harness emits them as `pending-human-review` placeholders and never fills
  them in.
- A release candidate fails when any pilot discovers silent loss, cross-project
  leakage, unverifiable final output or an unrecoverable revision.
