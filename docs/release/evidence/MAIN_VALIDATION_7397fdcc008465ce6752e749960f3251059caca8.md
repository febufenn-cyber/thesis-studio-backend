## Regression found and fixed

One merged-system defect surfaced and was fixed on this branch (not by
weakening any rule):

- `tests/test_release_candidate_hardening.py` — `test_production_requires_clamav`
  and `test_production_accepts_configured_clamav` failed on any developer
  machine with an untracked `.env` containing `DEBUG=true`: the test's
  `Settings(...)` construction read the ambient env file for fields it did not
  pass, so the production-safety validator raised on the DEBUG rule before the
  ClamAV rule. CI passed only because it has no `.env`. Fix: the test fixture
  now constructs `Settings(_env_file=None, …)`, making the assertion hermetic.
  The production-safety validator itself is unchanged. Suite result after fix:
  173/0/0/0.
